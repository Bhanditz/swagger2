Goals for the 2.0 spec.

Better names

A chance to improve the names of fields in the JSON.

API design-first workflow

A human-friendly YAML syntax and editor make it easy to create an API spec from scratch.

Extending the metadata

SLA information and vendor extensions bring a new level of flexibility to namespacing.

Best-of-breed tooling

Code generation and Swagger UI will get a major refresh following the new spec.

What is the timeframe?

The formation of a working group was announced by Tony Tam at Gluecon on May 22, 2014. The goal is to have a draft spec in June to be finalized in July.

Who is involved?

Reverb is convening the group with Apigee as a founding member. Join the group, and add your organization to this list!
